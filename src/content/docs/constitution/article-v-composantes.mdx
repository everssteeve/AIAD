## Article V — Les Composantes du Projet

Le projet AIAD est constitué de trois composantes interdépendantes. Chacune a un rôle distinct et une relation définie avec les autres.

### AIAD — Le Framework Organisationnel

AIAD définit *qui fait quoi* dans une équipe qui développe avec des agents IA. Il formalise cinq responsabilités (Product Manager, Product Engineer, Agents Engineer, QA Engineer, Tech Lead), quatre boucles itératives (Planifier, Implémenter, Valider, Intégrer), et les artefacts qui structurent la mémoire de l'équipe.

AIAD répond à la question : *comment s'organise-t-on pour que l'intention humaine guide l'ensemble du processus de développement ?*

### SDD Mode — Le Framework de Développement

SDD Mode (Spec Driven Development Mode) définit *comment on développe* avec des agents IA. Il formalise le cycle spec-first : Intent Statement → SPEC → Execution Gate → Développement agent → Validation → Drift Lock. Il introduit le Spec Quality Score, l'Anti-Drift Check, et le Context Budget comme outils opérationnels.

SDD Mode répond à la question : *comment s'assure-t-on que ce qui est construit reflète fidèlement ce qui était voulu ?*

### ALIS — Le Mécanisme d'Évolution

ALIS (système de mise à jour cyclique) définit *comment le framework lui-même évolue*. Il est le mécanisme constitutionnel du projet : il garantit qu'AIAD et SDD Mode sont des frameworks vivants, nourris par l'expérience terrain, évoluant de façon structurée et transparente.

ALIS répond à la question : *comment un framework reste-t-il en vie sans dériver de ses valeurs fondatrices ?*

La relation entre les trois composantes est hiérarchique dans les valeurs, collaborative dans les pratiques. AIAD pose le cadre organisationnel dans lequel SDD Mode opère. ALIS fait évoluer les deux selon les données collectées et les décisions humaines.
