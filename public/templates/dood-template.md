# Definition of Output Done (DoOD)

## Critères Techniques

- [ ] Conventions de code respectées
- [ ] Linting OK (aucune erreur)
- [ ] Types complets (pas de any/unknown non justifié)
- [ ] Tests unitaires passent
- [ ] Tests d'intégration passent

## Critères de Sécurité

- [ ] Scan de sécurité passé
- [ ] Pas de secrets dans le code
- [ ] Validation de toutes les entrées utilisateur
- [ ] Authentification/autorisation vérifiée

## Critères de Performance

- [ ] Budgets de performance respectés
- [ ] Queries optimisées
- [ ] Pas de N+1 queries

## Critères Fonctionnels

- [ ] SPEC respectée intégralement
- [ ] Acceptance criteria validés
- [ ] Cas limites gérés

## Critères de Déploiement

- [ ] Build réussit sans erreur
- [ ] Déployé en staging
- [ ] Smoke tests OK

## Critères de Review

- [ ] Code review faite et approuvée
- [ ] QA a validé les fonctionnalités
- [ ] Documentation mise à jour
